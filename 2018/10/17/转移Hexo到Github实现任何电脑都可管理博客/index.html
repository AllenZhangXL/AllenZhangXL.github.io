转移Hexo到Github实现任何电脑都可管理博客

Posted on 2018-10-17

我们已经通过Github，Hexo完成了Hexo博客的搭建，但是当我们更换电脑，或者想用其他电脑管理博客的时候，就会比较麻烦；又或者我们Hexo本地文件丢失之后，整个Hexo生成网站的原始文件就都会丢失。
所以，在看了廖雪峰老师的Git讲解之后,总和知乎大神的示范，决定写这个教程，帮助和我一样的小白把Hexo部署到Github上。

废话不多说，直接上步骤：

  1. 在原电脑上操作，给username.github.io博客仓库创建hexo分支，并且设置为默认分支。
  2. 如果未给你的github账号添加过当前电脑的SSH Key，需要创建SSH Key并且添加到Github账号上。
  3. 在当前电脑，任意一个位置，我是在桌面创建了一个test文件夹，执行命令 ： git clone git@github.com:username/username.github.io.git 把仓库conle到本地。
  4. 显示文件夹中的隐藏文件，删除刚才克隆的文档中，除了.git文件夹以外的所有内容。
  5. 命令行 cd 到 clone的仓库， git add -A git commit -m "--" git push origin hexo,把刚刚删除之后的本地仓库更新到远程仓库，刷新下github端博客hexo分支，应该已经清空了。
  6. 将上述.git文件夹复制到本地博客根目录下（就是有 themes，source等文件夹的那个目录下），到此我们就将本地博客目录和github的hexo分支相连了。
  7. 将博客目录下 themes文件夹下每个主题文件夹里的.git, .gitignore 删掉。
  8. cd 到博客目录， 输入命令 git add -A git commit -m "--" git push origin hexo 将本地博客目录同步到远程仓库的hexo分支。
    这里要注意，如果没有执行第7步操作，可能导致，同步失败。
    到此，我们在原电脑上的操作就结束了。
  9. 在新电脑上，我们先将新电脑的环境配置好，安装git、node.js、hexo、ssh key 也要创建添加好。
  10. 选好博客安装的目录，执行命令 git clone git@github.com:username/username.github.io.git.
  11. cd 到博客目录，npm install hexo g && hexo s 安装依赖，生成和启动博客服务，正常的话就可以打开浏览器 localhost:4000 就可看到博客了。
  12. 之后无论在哪台电脑更新，提交博客，依次执行，git pull git add -A git commin -m "--" git push origin hexo hexo clean && hexo g && hexo d 即可。
